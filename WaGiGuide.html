Introduction

Our application consists of 4 pages: Home, Play, Login and Dashboard.

Home Page

The Home page it's the first page that will be seen by our users. Here, you can find out answers to the following questions "What is WaGi?", "Why use WaGi?" and "How to use WaGi?".

Three animated bubbles with What?, Why?, How? are displayed to help users to explore the home page sections, and if he will click on them, he will be navigated to the answers.

From here, we can navigate to the other pages of the applications using the green navigation bar with white buttons from the top-right corner. In order to use the Play and Dashboard pages, first we have to log us in with an Stack Exchange account.

Login Page

Here our user can select the questions topic, by choosing with which Stack Exchange account he wants to play. Our users can play with accounts from Stack Overflow, Mathematics, Super User, Ask Ubuntu, Unix & Linux, Arqade, Webmasters, Blender and Cross Validated. The important thing is that all the responses from this game will be posted using the user account so... take care of your account.

If the user want to change the account, he can press on the logout button from the navigation bar and after that he can press on login and chose anoter account.

Play Page

The Play page is containing the actual game. Here we can see an Playground made of trees, bushes, hills and the skye with clouds. Over the clouds the we have displayed an text bubble with the question the user must answer. If the question text is too big, the user can scroll it to see the rest of it. After some seconds, in the forest will appear some deers, with different speeds and sizes (depending of how far are positioned). They will run from left to right or the other way and will try to make the user job harder. Each deer represents an answer for the displayed question and if the user moves the mouse cursor over them, another text bubble will appear neer the deer and will display the answer text. If the answer text it's too big to fit in the text bubble, the user is able to use the scroll over the deer to see the rest of the answer. One deer is more special from the others, because it display the following text "Try out to answer with your own answer!" which means if the user choose to hit this deer (click on it), another text bubble will appear under the question, with an text area where he can edit his own answer. Also, with the new text bubble, 2 buttons will appear Cancel and Submit that will be used to submit the new answer or to cancel it and to make the text area disappear.

An hit on a normal deer (click on it) means that the user want to upvote that answer, and the question will be changed and an notification will appear on the top of the page and will notify the user if the answer was the good one, or was a bad one, or other error messages like if he does not have enought reputation points. Also if the user hit the custom answer deer, and he submit an new answer, the question will be changed and an notification message will appear to notify him if the answer was posted and if it need some changes.

Dashboard Page

On the Dashboard page our users can see their account status regarding of the number of total answers, accepted answers, a badge for all of the user contributions on Stack Exchange and some information stored in our database like, accepted answers through our app, the success ratio and top of the favorite topics. All those topics are interactive so when the user will move the mouse over them more information will appear.

Also here we can see a very nice animation with a paper folded boat floting over some waves .
